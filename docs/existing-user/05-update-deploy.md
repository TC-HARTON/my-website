# サイト更新と再デプロイ

## このステップで行うこと
Site Builderで変更した設定をHTMLに反映し、品質チェック後にデプロイ（公開）します。

## 更新からデプロイまでの流れ
既存サイトの更新は、以下の5ステップで行います：

1. **設定を変更する** — 事業者情報・サービス・FAQ・SEOなどを編集
2. **設定反映（Inject）** — 変更内容をHTMLファイルに書き込む
3. **CSSビルド** — Tailwind CSSを再コンパイル
4. **SPECチェック** — サイト品質を検証（FAILが0件であること）
5. **デプロイ** — Gitにコミット＆プッシュ

## 各ステップの詳細

### Step 1: 設定変更
ダッシュボードやページタブで設定を変更します。変更はまだHTMLには反映されません。

### Step 2: 設定反映（Inject）
1. 「検証・デプロイ」タブを開く
2. 「設定反映 (Inject)」ボタンをクリック
3. Site Builderが各HTMLファイルのmetaタグ・JSON-LD・OGPなどを更新

**ポイント：** InjectシステムはHTMLに対して「最小差分」で変更を適用します。あなたが独自に追加したカスタムコード（独自のJavaScript、スタイル、HTML構造など）はそのまま保持されます。

### Step 3: CSSビルド
1. 「CSSビルド」ボタンをクリック
2. Tailwind CSSが再コンパイルされます

新しいTailwindクラスを使った場合（HTMLに新しいクラス名を追加した場合）は、CSSビルドが必要です。設定変更のみの場合は省略できることもあります。

### Step 4: SPECチェック
1. 「SPECチェック」ボタンをクリック
2. サイト全体の品質チェックが実行されます
3. 結果が表示されます：
   - **PASS** — 問題なし
   - **WARN** — 注意（デプロイ可能）
   - **FAIL** — 修正必要（デプロイ前に修正してください）
4. **FAILが0件** になるまで修正を繰り返す

### Step 5: デプロイ
1. SPECチェックがすべてPASS/WARNであることを確認
2. 「デプロイ」ボタンをクリック
3. Site Builderが自動で以下を実行：
   - `git add` — 変更ファイルをステージング
   - `git commit` — コミットメッセージを自動生成
   - `git push` — リモートリポジトリにプッシュ

## よくある質問

### Q. カスタムHTMLは上書きされませんか？
Injectシステムは管理対象タグ（`<title>`、`<meta>`、JSON-LDスクリプトなど）のみを更新します。独自に追加したHTML・JavaScript・CSSはそのまま保持されます。

### Q. SPECチェックでFAILが出たらどうすればいい？
FAIL項目の詳細を確認し、該当する設定を修正してください。よくある原因は、必須項目（事業者名・ドメインなど）の未入力や、meta descriptionの文字数不足です。

### Q. 変更を元に戻せますか？
Gitで管理されているため、デプロイ前であれば変更を破棄できます。デプロイ後も、Gitの履歴から以前の状態に戻すことが可能です。

### Q. どのくらいの頻度でデプロイすればいいですか？
内容を更新したら、その都度デプロイすることをおすすめします。こまめにデプロイすることで、変更履歴が残り、問題が起きた場合の切り戻しも容易になります。

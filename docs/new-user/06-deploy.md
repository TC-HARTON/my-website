# サイト公開・デプロイ方法

## このステップで行うこと
生成したサイトをインターネットに公開します。

## デプロイの仕組み
Site Builderは Git + ホスティングサービス でデプロイします：
1. サイトファイルをGitでコミット
2. GitHubにプッシュ
3. ホスティングサービスが自動的にサイトを公開

## 対応ホスティングサービス
- **Cloudflare Pages** — 高速CDN・無制限帯域・Workers連携（推奨）
- **GitHub Pages** — 無料・GitHub統合
- **Vercel** — Jamstack特化・自動プレビュー
- **Netlify** — フォーム・認証付き

## 事前準備
1. GitHubアカウントを作成
2. GitHubにリポジトリを作成
3. ホスティングサービスでGitHubリポジトリを連携
4. 初期設定 > Git・デプロイ でブランチ等を設定

## 操作手順
1. 「検証・デプロイ」タブを開く
2. SPECチェックを実行（0 FAILを確認）
3. 「デプロイ」ボタンをクリック
4. 自動で git add → commit → push が実行されます

## よくある質問

### Q. GitやGitHubが分からない
デプロイにはGitの基本知識が必要です。初回セットアップのみ少し手間がかかりますが、以降はワンクリックで更新できます。

### Q. デプロイ後にサイトが表示されない
ホスティングサービスの設定を確認してください。ビルドコマンドは不要（静的HTMLをそのまま配信）です。

### Q. 独自ドメインを使いたい
各ホスティングサービスのカスタムドメイン設定で追加できます。DNS設定が必要です。

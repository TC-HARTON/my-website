# Git・デプロイの設定

> 作ったサイトをインターネットに公開する方法を決めます

## はじめに

「デプロイ」とは、パソコンで作ったサイトをインターネットに公開することです。
GitHubというサービスを使うと、ファイルをアップロードするだけで自動的にサイトが公開されます。

**公開の準備ができるまで設定しなくても大丈夫です。まずはパソコン上でサイトを確認しながら作りましょう。**

## GitHubとは

[GitHub](https://github.com/) は、ファイルの管理・共有ができる無料サービスです。
サイトのファイルをGitHubにアップロードすると、連携したホスティングサービスが自動的にサイトを公開してくれます。

### アカウント作成
1. [https://github.com/](https://github.com/) にアクセス
2. 「Sign up」からメールアドレスで無料登録
3. リポジトリ（ファイル置き場）を作成

## デプロイ先の選択

GitHubと連携してサイトを公開できるサービスです:

| サービス | 特徴 | おすすめ度 |
|---------|------|----------|
| [Cloudflare Pages](https://pages.cloudflare.com/) | 最速・無料SSL・全世界配信 | ★★★ |
| [GitHub Pages](https://pages.github.com/) | 最もシンプル・設定不要 | ★★★ |
| [Netlify](https://www.netlify.com/) | ドラッグ&ドロップも可 | ★★☆ |
| [Vercel](https://vercel.com/) | 高機能・プレビュー機能 | ★★☆ |

## 設定手順

### ローカルで始める場合（デプロイなし）
1. Site Builderでサイトを生成
2. ブラウザでプレビュー確認
3. 満足したら後からデプロイ設定を追加

### GitHub + Cloudflare Pagesの場合
1. GitHubにリポジトリを作成
2. Site Builderの「デプロイ設定」→ プラットフォームで「cloudflare-pages」を選択
3. ブランチ名は「main」のまま
4. 「デプロイ実行」でGitHubにプッシュ → Cloudflareが自動公開

## よくある質問

**Q: GitHubがわからなくても大丈夫ですか？**
A: はい。まずはローカルでサイトを作り、公開準備ができたら設定しましょう。Site Builderが公開作業を自動化します。

**Q: 費用はかかりますか？**
A: GitHub、Cloudflare Pages、GitHub Pages、Netlifyは全て無料枠で十分です。

**Q: サイトを更新するたびに手動で公開が必要ですか？**
A: いいえ。Site Builderの「デプロイ実行」ボタンひとつで更新が自動反映されます。

**Q: 後からデプロイ先を変更できますか？**
A: はい。設定画面からいつでも変更できます。

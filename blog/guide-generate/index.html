サイト生成の手順 — ワンクリックで構築

公開: 2026-04-12 著者: 大内 達也
Site Builder ガイド #Site Builder #初心者向け #サイト生成

サイト生成の前提条件

サイトを生成するには、以下の4つが最低限必要です。すべてSite Builderの設定画面から入力できます。

  1. サイトフォルダの保存先パスが設定されていること
  2. Anthropic Claude API キー（sk-ant-api... 形式）が設定・接続テスト済みであること
  3. 事業名（会社名または屋号）が入力されていること
  4. ドメイン（サイトのURL）が設定されていること

これらの設定が揃っていれば、ボタンひとつでサイト全体を自動生成できます。

生成されるファイル一覧

ワンクリックの生成で、以下のファイルとフォルダが自動的に作成されます。

  • index.html — トップページ。事業情報・サービス・FAQなどが配置されます
  • サービス個別ページ — 登録したサービスごとに詳細ページが生成されます
  • privacy/index.html — プライバシーポリシーページ
  • profile/index.html — E-E-A-T著者プロフィールページ
  • 404.html — カスタムエラーページ（ページが見つからない場合に表示）
  • thanks.html — フォーム送信完了後のサンクスページ
  • tailwind.config.js — Tailwind CSSフレームワークの設定ファイル
  • package.json — プロジェクトの依存パッケージ管理ファイル

自動実行される処理

ファイル生成後、以下の処理がバックグラウンドで自動実行されます。手動でのコマンド操作は一切不要です。

  1. npm install — Tailwind CSSなど必要なパッケージを自動インストール
  2. git init — Gitリポジトリの初期化（バージョン管理の準備）
  3. CSSビルド — Tailwind CSSをコンパイルして最終的なスタイルシートを生成

生成後の確認ステップ

  1. SPECチェック — spec-checker (SPEC v2.3 + GEO/LLMO 統合版) が 313 項目 + G-1〜G-6 の生成エンジン最適化を自動検証します
  2. ローカルプレビュー — ブラウザでサイトの表示を確認します
  3. デプロイ — 問題なければボタンひとつで世界に公開します

生成時に何が起こっているか（裏側）

ボタン一発の裏側では、以下の処理が自動で走っています。

  • 3法規 40KB を埋込 — SPEC v2.3 / GOOGLE-STANDARDS / GEO-STANDARDS がシステムプロンプトに常駐し、Claude Opus 4.6 が3法規に準拠してコンテンツ生成
  • モデル分業ルーティング — routingMode=balanced の場合、サイト構造設計は Opus、HTMLコーディングも Opus、検証は Haiku と自動分業
  • プロンプトキャッシュ — 2回目以降の生成は 最大 85% のトークン費用削減（Anthropic公式機能）
  • BYO-Key 保護 — APIキーは AES-256 でローカル暗号化、アプリ運営側を経由しません

よくある質問（FAQ）

生成にはどのくらい時間がかかりますか？

AIの応答速度と登録サービス数にもよりますが、通常1〜5分程度で全ファイルの生成が完了します。進捗は画面上にリアルタイムで表示されます。

生成後にファイルを手動で編集しても大丈夫ですか？

はい、生成されたHTMLやCSSは自由に編集できます。ただし、Site Builderから再度サイト生成を行うとファイルが上書きされるため、カスタマイズ内容はバックアップを取っておくことをおすすめします。

生成に失敗した場合はどうすればいいですか？

まず Claude API キーが有効で、Anthropic アカウントにクレジット残高があることを確認してから再実行してください。エラーメッセージが表示される場合は、その内容をもとに原因を特定し対処できます。

まとめ

サイト生成は、必要な情報を入力してワンクリックするだけで完了します。HTMLからCSS、構造化データ、プライバシーポリシーまですべて自動生成。Site Builderなら、プログラミング知識ゼロでもプロ品質のビジネスサイトが数分で完成します。

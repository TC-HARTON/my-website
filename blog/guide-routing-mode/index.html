routingMode完全ガイド — quality/balanced/economy

公開: 2026-04-16 著者: 大内 達也
Claude API #Claude #routingMode #モデル分業

結論 — balanced モードで費用 -65% / 品質そのまま

本記事で判明する数値: quality モード $2.05 → balanced モード $0.72（-65%）、economy モード $0.28（-86%）。S-RANK保証は quality/balanced のみ、economy は検証用。月5サイトで年間 ¥140,000 以上の差になります。

「複数モデルの分業（Mixture of Experts）は、タスク特性に応じた最適モデル選択で品質を維持しつつ費用を大幅削減できる」

— Aggarwal et al. Generative Engine Optimization, KDD 2024

なぜ validate は Haiku で十分か

SPEC v2.3 の 649項目検証は「パターンマッチング」と「構造化データの形式検査」が主で、深い推論を必要としません。Haiku 4.5 は正規表現的タスクで Opus と同等の精度を持ち、費用は 1/15 です。このため balanced モードでは validate を Haiku に委譲しています。

サイト生成で Opus を外してはいけない理由

generate/repair タスクは 3法規40KB を参照しながら業種別最適化を行う複合推論で、Sonnet 以下では以下の問題が発生します。

  • G-6 位置最適化: 上位30%にエビデンス配置する位置制御の精度低下
  • Authoritative Tone: 権威的論調と親しみやすさのバランス失調
  • JSON-LD の整合性: Organization / Person / Product 複数スキーマの相互整合

モード推奨マトリクス

  • quality: 最重要案件（大手クライアント・ブランドサイト）、月1〜2本
  • balanced: 通常業務の推奨（中小企業サイト・LP・ブログ）、月3〜10本
  • economy: 試作・検証用途のみ（本番環境では非推奨）

よくある質問（FAQ）

モード切替は生成後にできますか？

はい。Site Builder 設定画面から即時切替可能で、次回生成から反映されます。既存サイトの再生成にも有効です。

economy で作ったサイトは品質劣化しますか？

A クラス想定のため S-RANK 基準を満たさない可能性があります（FAIL 2〜5件の発生）。本番公開前に必ず quality/balanced で再生成してください。

プロンプトキャッシュとの併用は？

はい、プロンプトキャッシュは全モードで自動適用されます。balanced モード + キャッシュで実質 -85% が実現します（詳細は キャッシュ解説）。

まとめ

routingMode は quality / balanced / economy の3モードで、実測費用は $2.05 / $0.72 / $0.28。通常業務は balanced 推奨（-65%、S保証維持）、最重要案件のみ quality を選択するのが最適解です。

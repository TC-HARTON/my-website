API課金が怖い人へ — 上限設定と安全装置の全て

公開: 2026-04-16 著者: 大内 達也
Claude API #Site Builder #Claude API #費用 #安全設定

結論 — 「使いすぎて破産」は物理的に不可能

本記事で判明する事実: Anthropic コンソールで月額上限（Usage Limit）を設定すれば、上限到達時にAPIは自動停止します。$5（約¥750）に設定すれば LP 約12枚分で止まり、それ以上の請求は発生しません。さらに Site Builder の routingMode 切替で1クリックでコスト半減も可能です。

「Usage Limits を設定すると、月間利用額が上限に達した時点でAPIリクエストは自動的に拒否される。意図しない超過課金を防ぐための安全装置として機能する」

— Anthropic 公式ドキュメント — Initial Setup

Claude MAX プランと API は別物です

まず重要な前提を整理します。Claude MAX プラン（$100/月）とAnthropic API（従量課金）は完全に別のサービスです。

Claude MAX プランAnthropic API
提供先claude.ai（チャットUI）console.anthropic.com
課金方式月額固定（$100 or $200）従量課金（使った分だけ）
用途ブラウザでの対話外部アプリからの呼び出し
Site Builder❌ 使用不可✅ これが必要
キー形式なし（ログインで使用）sk-ant-api03-...

MAX プランに加入していても、Site Builder を使うには別途 API キーの取得（無料）が必要です。逆に言えば、MAX プランに入る必要はありません。

安全装置① — 月額上限（Usage Limit）の設定

console.anthropic.com/settings/limits から月額上限を設定できます。

  1. Anthropic Console にログイン
  2. Settings → Plans & Billing → Usage Limits
  3. Monthly Limit を任意の金額に設定（例: $5）
  4. 上限到達でAPIが自動停止（追加請求なし）

安全装置② — routingMode でコスト制御

Site Builder の設定画面から routingMode を切り替えるだけで、同じ作業でも費用が大幅に変わります。

安全装置③ — プロンプトキャッシュで自動節約

Site Builder は3法規（SPEC v2.3 + GOOGLE-STANDARDS + GEO-STANDARDS / 計40KB）をシステムプロンプトに埋込みますが、Anthropic のプロンプトキャッシュにより2回目以降は入力コスト 90% OFFが自動適用されます。ユーザー側の操作は不要で、使うほど安くなる仕組みです（詳細解説）。

よくある質問（FAQ）

上限に達したらサイトは消えますか？

いいえ。生成済みのサイトはローカルに保存されており、APIが停止してもサイトの閲覧・公開には一切影響しません。新規生成・修正のみ翌月まで待つか、上限を引き上げてください。

クレジットカード登録は必要ですか？

API利用には支払い方法の登録が必要です。ただし初回 $5 無料クレジットが付与される場合があり、LP 約12枚分は無料で試せます。

Claude MAX プランに入っていれば API は不要ですか？

いいえ、別途必要です。MAX プランは claude.ai チャットUI の利用枠であり、Site Builder のような外部アプリからの接続には API キー（従量課金）が必要です。ただし API の費用は極めて低額（月¥数百程度）なので、MAX プランの有無に関わらず負担は軽微です。

為替変動で高くなりませんか？

1 USD = 140〜160円の変動幅で最大 ±10% 程度です。$5 上限設定なら ¥700〜800 の範囲内で、ブレは数十円レベルです。

まとめ

Claude API の従量課金は3重の安全装置で守られています。① 月額上限で自動停止、② routingMode で1クリック節約、③ プロンプトキャッシュで自動90%OFF。LP 1枚 ¥60 の世界で「使いすぎ」は起きません。まずは $5 上限で安心してお試しください。
